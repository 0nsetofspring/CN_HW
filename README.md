# 💬 CN_CHAT: AI 협업 기반 멀티 쓰레드 CLI 채팅 프로그램

> **"CLI 환경의 한계를 극복하는 사용자 중심(UX) 네트워크 프로그래밍"** <br>
> 본 프로젝트는 C++ 소켓 통신과 멀티 쓰레딩을 활용하여 다중 접속을 지원하는 채팅 서버/클라이언트 시스템입니다.<br>
> 단순 텍스트 전송을 넘어, 터미널 환경에서도 직관적이고 친화적인 사용자 경험을 제공하도록 설계되었습니다.<br>
> 26-1 Hanyang Univ. Summer Session - Computer Network 

<br>

## ✨ 핵심 기능 (Key Features)

- **다대다 및 1:1 통신:** 전체 브로드캐스팅(광장) 및 특정 유저를 향한 귓속말(유니캐스트) 지원.
- **CLI UX 극대화:** 터미널 색상 코드를 통한 텍스트 테마 분리, 접속 타임스탬프, 웰컴 배너(MOTD).
- **편의 기능 (Onboarding):** 이모티콘 매크로, 단축어 저장, 귓속말 간편 답장(`/wr`)/최근 내역(`/wl`), 명령어 오타 교정 제안.
- **백엔드 고도화:** 서버 메모리를 활용한 대화 검색, 서버 측 비속어 필터링(클린 봇), 도배 방지(Rate Limiting), 닉네임 중복 방지.
- **미니 챗봇 통합:** 주사위 굴리기, 랜덤 유저 추첨, 실시간 투표(시작자만 종료 가능) 등 인게임 액션 지원.

## 🚀 시작하기 (Getting Started)

### 1. 빌드 (컴파일)
본 프로그램은 리눅스(Linux) 환경 및 GCC 컴파일러를 기준으로 작성되었습니다. 멀티 쓰레드 동작을 위해 `-pthread` 옵션이 필수입니다.

```bash
# 서버 빌드
$ g++ -o server multi_server.cc -pthread

# 클라이언트 빌드
$ g++ -o client client.cc -pthread

```

### 2. 실행 방법

```bash
# 서버 실행 (포트번호 지정)
$ ./server 8080

# 클라이언트 접속 (서버IP, 포트번호, 초기 닉네임 지정)
$ ./client 127.0.0.1 8080 User1

```

## 📖 사용자 가이드 (User Commands)

채팅창(터미널)에서 슬래시(`/`)를 활용하여 다양한 기능을 즉시 사용할 수 있습니다.

| 기능 분류 | 명령어 예시 | 설명 |
| --- | --- | --- |
| **도움말/설정** | `/help` | 전체 명령어 및 사용법 출력 |
|  | `/clear` | 터미널 채팅 화면 초기화 |
|  | `/name [새이름]` | 자신의 닉네임 변경 (전체 유저에게 알림) |
| **단축어** | `/alias [단축] [원본]` | 자주 쓰는 문장 저장 (예: `/alias ㅂㅂ 수고하셨습니다`) |
|  | `/aliases` | 현재 저장된 내 단축어 목록 확인 |
| **소통/조회** | `/list` | 접속 중인 전체 유저 목록 및 유지 시간 확인 |
|  | `/w [대상] [할말]` | 특정 유저에게만 귓속말 전송 |
|  | `/wr [할말]` | 마지막으로 받은 귓속말 발신자에게 바로 답장 |
|  | `/wl [N]` | 최근 귓속말 N개(기본 10개) 확인 (내 화면 전용, 서버엔 기록 안 남음) |
|  | `/search [단어]` | 최근 대화 기록 중 특정 단어가 포함된 내역 검색 |
|  | `@닉네임 [할말]` | **[멘션]** 대상 유저의 화면 강조 및 비프음 발생 |
| **미니 챗봇** | `/bot` | 봇이 제공하는 전체 명령어 가이드 출력 |
|  | `/bot dice` | 1~100 사이 랜덤 숫자 생성 및 전체 방송 |
|  | `/bot random_user` | 현재 접속 중인 유저 중 한 명 랜덤 추첨 |
|  | `/bot poll [A] [B]` | A와 B 중 선택하는 실시간 투표 진행 |
| **종료** | `/quit` (또는 `q`) | 서버와 연결 안전하게 종료 |

*(💡 Tip: 채팅창에 `(하트)`, `(따봉)`을 입력하면 자동으로 ❤️, 👍 이모티콘으로 변환되어 전송됩니다.)*

## ⚙️ 시스템 아키텍처 및 통신 프로토콜 (For Grader)

패킷(메시지)의 앞단에 태그(Tag)를 부착하여 클라이언트와 서버가 서로의 의도를 명확히 파싱(Parsing)하도록 설계되었습니다.
<br><br>

### 통신 프로토콜 명세 (Protocol Specification)

| 방향 | 태그(Prefix) | 페이로드(Payload) 예시 | 시스템 처리 방식 |
| --- | --- | --- | --- |
| `C ➡️ S` | `[CHAT]` | `[CHAT]안녕하세요` | 서버가 비속어 필터링 후 전체 브로드캐스팅 |
| `C ➡️ S` | `[REQ:NAME]` | `[REQ:NAME]새닉네임` | 해당 소켓의 닉네임 상태 업데이트 및 알림 방송 |
| `C ➡️ S` | `[REQ:WHISPER]` | `[REQ:WHISPER]대상\|할말` | 대상 닉네임의 소켓 FD를 탐색하여 1:1 유니캐스트 (최초 `|` 1개만 분리 파싱) |
| `C ➡️ S` | `[REQ:BOT]` | `[REQ:BOT]dice` / `[REQ:BOT]poll A\|B` / `[REQ:BOT]vote A` / `[REQ:BOT]poll_end` | 서버 내부 봇 로직 실행 후 결과 브로드캐스팅 |
| `C ➡️ S` | `[REQ:LIST]` | `[REQ:LIST]` | 접속자 배열을 순회하여 리스트 반환 |
| `C ➡️ S` | `[REQ:SEARCH]` | `[REQ:SEARCH]과제` | 대화 기록 버퍼 탐색 후 결과 반환 |
| `S ➡️ C` | `[SYS]` | `[SYS]OOO님 입장` | 클라이언트 단에서 파란색 시스템 테마로 렌더링 |
| `S ➡️ C` | `[ERR]` | `[ERR]잘못된 명령어` | 클라이언트 단에서 붉은색 에러 테마로 렌더링 |
| `S ➡️ C` | `[RES:LIST]` | `[RES:LIST]user1,120\|user2,45` | `/list` 응답. 닉네임,접속유지초 를 `|`로 join |
| `S ➡️ C` | `[RES:SEARCH]` | `[RES:SEARCH]닉네임: 내용` | `/search` 응답. 채팅 내용에 `|` 제약이 없어 여러 결과를 한 줄로 합치지 않고, 매칭 건수만큼 메시지를 반복 전송. 결과 없으면 빈 payload 1개 |
| `S ➡️ C` | `[WHISPER]` | `[WHISPER]보낸이\|할말` | 귓속말 수신 측에게만 전달. 클라이언트가 비프음(`\a`) 처리 |
| `S ➡️ C` | `[WHISPER_SENT]` | `[WHISPER_SENT]받는이\|할말` | 귓속말을 보낸 본인에게 전송 확인용으로 에코 |


<br>

### 프로토콜 제약사항 (Design Constraints)

- **닉네임:** `|` 문자와 공백 사용 금지, 최대 20자. (구분자 충돌 방지 — WHISPER/LIST payload가 `|`로 필드를 나누기 때문)
- **WHISPER 파싱:** payload를 `|` 기준 최초 1회만 split. 할말 내용에 `|`가 포함돼도 안전하게 대상만 분리됨.
- **MENTION(`@닉네임`) 매칭:** 부분 문자열 오탐 방지를 위해 `@닉네임` 뒤에 공백/문장부호/줄끝이 와야만 매치로 인정 (예: `@길동아`가 `@길동`에 오탐되지 않도록).
- **POLL 생명주기:** `/bot poll A B` 로 시작 → 투표는 `/bot vote A`(또는 B) → 30초 타임아웃 또는 `/bot poll_end` 수동 종료 중 먼저 발생하는 시점에 결과 브로드캐스트. 진행 중 poll이 있으면 새 poll 시작 요청은 `[ERR]`로 거부.
- **SEARCH 버퍼:** 대화 기록은 무제한 저장하지 않고 최근 200개만 유지하는 ring buffer로 관리 (메모리/검색 성능 보호).
- **동시성:** 클라이언트 목록(닉네임/접속시각 포함 구조체)과 대화 기록 버퍼, poll 상태는 모두 공유 자원이므로 각각 `pthread_mutex_t`로 보호. 원본 `clnt_socks[]` 배열은 Phase 2에서 구조체 배열로 교체하면서 mutex를 함께 도입.
- **닉네임 중복 금지:** 중복을 허용하면 `/w`, `@멘션`, `/list`가 어느 쪽을 가리키는지 모호해지므로, `/name` 처리 시 형식 검증 후 중복 여부도 확인(자기 자신으로 재등록은 허용).
- **도배 방지(Rate Limiting):** 접속(fd)당 2초에 15개 초과 메시지는 거부. 엔터 연타/매크로 같은 진짜 도배는 막되, 사람이 명령어 여러 개를 정상적으로 빠르게 입력하는 흐름은 막지 않도록 실제 테스트로 임계값을 조정함.
- **자기 자신 귓속말 금지:** `/w 내닉네임`은 `[ERR]`로 거부.
- **POLL 종료 권한:** `/bot poll_end`는 투표를 시작한 사람만 가능. 30초 자동 타임아웃은 시스템이 마감하는 것이라 이 권한 체크와 무관하게 항상 동작.
- **명령어 오타 교정:** `/`로 시작하는데 아는 명령어와 안 맞으면 Levenshtein 편집 거리로 가장 가까운 명령어를 로컬에서 제안하고 전송은 하지 않음(잘못된 텍스트가 채팅으로 새어나가는 것 방지). 외부 라이브러리 없이 순수 문자열 처리로 구현.

- [x] (선행) `clnt_socks[]` int 배열 → `Client{fd, nickname, join_time}` 구조체 배열 + `pthread_mutex_t clients_lock` 전환
- [x] 태그 기반 파싱 로직 (`[CHAT]`, `[REQ:NAME]`)
